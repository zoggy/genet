When specifying a chain, it is not possible to connect two ports of interfaces producing and consuming different file types.

However, it is possible ports producing a set of files of a given filetype to a port consuming a file of the same file type. This will result in running the operation containing the second port for each file produced by the first port.

Let's illustrate this by an example. Following , we define a second chain, to compute the average length of paragraphs. To do so, we need to:

  • upgrade the split-text tool to a new version, handling option -p to split text according to blank lines instead of spaces,
  • upgrade the average> tool to a new version, handling option -n indicating that input files contain numbers, and it is the average of these numbers that must be printed, and handling a -c option indicating to only count input characters, not compute average,
  • add new interfaces split-in-pars, average-of-set and text-length to use the new features in chains,
  • define a new chain implementing implementing the workflow.
